### 3.4 项目宪法（Constitution）

项目从零建立了 7 项核心原则的宪法约束体系，版本演进至 v1.3.3：

| 原则 | 内容 |
|------|------|
| I | 分层架构（6 层职责分离） |
| II | 统一响应格式（`{code, message, data, trace_id, cost, extra}`） |
| III | 可复制为模板（配置分离、禁止惰性建表） |
| IV | 英文代码产物（注释、commit message 英文；文档中文） |
| V | 测试覆盖（每个 handler/model 必须有测试，测试行数 > 生产代码行数） |
| VI | 错误及时抛出（Fail Fast，禁止静默吞错） |
| VII | 数据库表由用户创建（禁止应用自动建表，schema 由 SQL 脚本管理） |
